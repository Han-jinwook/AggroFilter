## 1. 개요 및 시스템 분리 배경 (Overview)

이 시스템은 어그로필터 메인 서비스와 철저히 분리되어, 대표님의 로컬 PC(또는 소형 서버)에서 단독으로 돌아가는 **'자동화 봇(Bot) 및 제어판'**입니다.

### 1.1 아키텍처 구조
- **위치**: 메인 프로젝트 Root 하위에 독립 폴더(`auto-marketer/`). Node.js 기반.
- **실행 방식**: 봇 스크립트 + 로컬 웹 서버(`http://localhost:3001`) = 대시보드 UI 제공.
- **DB 연동**: 메인 웹앱과 동일 DB. 봇은 Insert/Update 역할 집중. (의뢰자 식별: `user_id = 'bot'`)

### 1.2 전략 전환 배경 (2026-03-09)
- 모바일 앱 전략을 **PWA(조회/랭킹 중심)**로 선회 → 유저의 '조회(View)' 흥미를 극대화해야 함.
- **문제**: 기존 수집기가 유튜브 공식 '인기 급상승' 탭만 바라봐서, 점수 높은(착한) 샌님 영상만 쌓임.
- **목표**: 팩트체크가 시급한 렉카, 가짜뉴스 등 **어그로 점수가 높은(빨간/노란색) 자극적인 영상**을 집중 사냥하여 앱의 도파민과 활용도를 높인다.

## 4. [섹션 2] 어그로 키워드 사냥 (Type3, 신설) 🎯�

### 4.1 전술 개요: 그물을 넓게 던져라 (Keyword-based Wide Net)

유튜브 공식 트렌딩 탭 의존도를 낮추고, **음지에서 조회수를 빨아먹는 자극적인 영상을 직접 검색**해서 건져 올린다.

### 4.2 키워드 그룹 (대시보드에서 추가/수정 가능)

키워드는 **하드코딩하지 않는다**. 대시보드 옵션에서 그룹별로 실시간 편집 가능.

| 그룹명 | 초기 키워드 |
|--------|-----------|
| **일반/이슈/렉카** | 단독, 충격, 폭로, 경악, 논란, 소름, 근황, 결말, 진실, 숨겨진, 참교육, 분노 |
| **금전손해/사기/코인** | 폭락, 상폐, 먹튀, 전재산, 청산, 세력, 급등, 떡상, 사기, 피해자, 원금보장, 무조건, 폰지 |

### 4.3 수집 파이프라인

```
1. 키워드 그룹별 YouTube search/list API (24h 이내, order: viewCount)
2. 기본 필터링 (쇼츠 제외, 한국어, 중복, 채널 쿨타임)
3. 후보 수백개 확보
4. 자막 수집 (youtube-transcript)
5. ★ 오픈소스 AI 사전 스코어링 (어그로성 0~100점)
6. 상위 N개 또는 N% 이상만 선별
7. 유료 정식 AI 분석 → DB 저장
8. 결과 리포트 (대시보드)
```

### 4.4 오픈소스 AI 사전 스코어링 (비용 0원)

유료 AI 호출을 최소화하기 위한 **사전 필터 단계**. 수백 개의 후보 중 진짜 어그로인 것만 골라낸다.

- **모델**: **Qwen 2.5 7B** (Ollama 로컬 구동)
  - 한국어 이해도 7B급 최상급 (아시아 언어 특화)
  - 8GB RAM이면 구동 가능
  - 설치: `ollama pull qwen2.5:7b`
- **입력**: 영상 제목 + 자막 앞부분 (500자)
- **출력**: 어그로성 점수 0~100 (JSON)
- **프롬프트**: "이 유튜브 영상의 제목과 자막을 보고 어그로성(과장, 낚시, 선정성, 공포 조장) 점수를 0~100으로 매겨라. JSON으로 {score: number, reason: string} 반환."
- **비용**: 완전 무료 (로컬 추론)

### 4.5 옵션 (대시보드)

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| K | 키워드당 검색 개수 | 30 |
| 사전필터 컷라인 | 오픈소스 AI 점수 N점 이상만 유료 분석 | 60 |
| 유료 분석 일일한도 | 하루 최대 유료 분석 횟수 | 30 |
| 키워드 그룹 편집 | 그룹 추가/수정/삭제 | 대시보드 UI |

## 5. [섹션 3] 요주의 채널 딥다이브 (신설) 🕵️

### 5.1 전술 개요: 낚인 놈은 끝까지 판다 (Targeted Seed Deep Dive)

어그로를 끄는 채널은 정해져 있다. 한 번 어그로 점수가 높게 나온 채널(렉카 등)의 **진성 어그로 영상들을 집중 추가 수집**하여 데이터의 깊이를 더한다.

### 5.2 요주의 채널 자동 등록

- `t_analyses`에서 `f_clickbait_score >= N` (옵션) 인 채널을 자동 리스트업
- 대시보드에서 리스트 조회 + N값 조절 + 수동 추가/제거 가능

### 5.3 서칭 옵션

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| 어그로 임계값 | f_clickbait_score N점 이상 채널 등록 | 80 |
| 서칭 주기 | X일에 1회 해당 채널 순회 | 3일 |
| 분석 방식 A | 해당 기간 중 조회수 최고 영상 분석 | ✅ 기본 |
| 분석 방식 B | 당일 검색으로 걸린 최신 영상 분석 | 선택 |

### 5.4 파이프라인

```
1. DB에서 요주의 채널 리스트 조회 (f_clickbait_score >= N)
2. 각 채널 YouTube search/list (channelId 파라미터, 서칭 주기 내)
3. 방식 A: 조회수 최고 영상 선택 / 방식 B: 최신 영상 선택
4. 기분석 제외 필터
5. 유료 정식 AI 분석 → DB 저장
6. 결과 리포트 (대시보드)
```

## 8. 섹션 2 구현 완료 내역 (2026-03-12)

### 8.1 대시보드 UI
- **탭 추가**: "🎯 키워드 사냥" 탭 신설
- **3개 카드 구성**:
  - 📊 현황 카드: 오토모드, 실행 상태, 수동 실행 버튼
  - 🔑 키워드 관리 카드: 그룹 추가/수정/삭제, 활성화 토글
  - ⚙️ 설정 카드: K(키워드당 검색 개수), 사전필터 컷라인, 유료 분석 일일한도
- **결과 테이블**: 키워드별 수집 결과 최신 200개 표시

### 8.2 백엔드 API
- `GET /api/section2/status` - 상태 조회
- `POST /api/section2/automode` - 오토모드 토글
- `POST /api/section2/options` - 설정 저장
- `POST /api/section2/run` - 수동 실행 (runJobType3 연결)
- `GET /api/section2/results` - 수집 결과 조회
- `GET /api/keyword-groups` - 키워드 그룹 목록
- `POST /api/keyword-groups` - 키워드 그룹 추가
- `PATCH /api/keyword-groups/:id` - 키워드 그룹 수정
- `DELETE /api/keyword-groups/:id` - 키워드 그룹 삭제

### 8.3 수집 로직
- **YouTube 검색**: `collectSection2` 함수 (youtube-collector.js)
  - DB에서 활성 키워드 그룹 로드
  - 각 키워드로 YouTube 검색 (24시간 이내, 조회수 순)
  - 기본 필터링 (쇼츠 제외, 한국어, 중복, 채널 쿨타임)
- **사전 스코어링**: `aggro-prescorer.js`
  - Ollama + Qwen 2.5 7B 로컬 추론
  - 어그로성 점수 0~100 산출 (비용 0원)
  - 컷라인 이상만 유료 분석 대상 선정
- **유료 분석**: `runJobSection2` 함수 (job.js)
  - 자막 수집 → 사전 스코어링 → 컷라인 필터 → 일일한도 적용 → 유료 AI 분석
  - 결과 DB 저장 (`f_requester_id = 'bot-section2'`)

### 8.6 함수 네이밍 규칙 (2026-03-12 정리)
- **섹션 구분 명확화**: Type3 → Section2로 변경
  - `collectType3` → `collectSection2`
  - `collectType3Only` → `collectSection2Only`
  - `runJobType3` → `runJobSection2`
- **이유**: 섹션 번호와 타입 번호 혼동 방지, 코드 가독성 향상

### 8.4 데이터베이스
- `bot_aggro_keywords` 테이블: 키워드 그룹 관리
  - `id`, `group_name`, `keywords` (text[]), `is_active`, `created_at`, `updated_at`
- `t_analyses` 테이블: 분석 결과 저장
  - 섹션2 수집 영상은 `f_requester_id = 'bot-section2'`로 구분
  - `f_keyword` 컬럼에 검색 키워드 저장

### 8.5 설정 파일
- `auto-marketer/data/section2_options.json`:
  - `keywordSearchCount`: 키워드당 검색 개수 (기본값: 30)
  - `prefilterCutline`: 사전필터 컷라인 (기본값: 60)
  - `dailyAnalysisLimit`: 유료 분석 일일한도 (기본값: 30)
