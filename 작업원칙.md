# AggroFilter 프로젝트 작업 원칙

이 문서는 Cascade와 협업하여 AggroFilter 프로젝트를 진행하기 위한 핵심 규칙을 정의합니다.

## 🚀 핵심 원칙

**"선(先) 분석 및 계획 제시, 후(後) 컨펌 및 실행"**

모든 작업은 아래의 절차를 반드시 준수한다.

1.  **분석 (Analyze):** 사용자의 요구사항과 현재 코드베이스를 분석한다.
2.  **계획 제시 (Propose):** 분석 결과를 바탕으로 구체적인 작업 계획(Plan)과 예상 변경 사항을 사용자에게 명확하게 제시한다.
3.  **컨펌 (Confirm):** 사용자의 승인 또는 수정 피드백을 받는다.
4.  **실행 (Execute):** 컨펌된 계획에 따라서만 실제 코드 수정, 파일 생성 등의 작업을 수행한다.

--- 
*이 원칙은 프로젝트의 안정성을 확보하고, 예상치 못한 변경을 방지하며, 명확한 방향성을 유지하기 위해 수립되었습니다.*
