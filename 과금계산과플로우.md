# 🪙 어그로필터 정밀 동적 과금 시스템 가이드 (v1.3)
> **작성 일시**: 2026-05-11 23:25 (KST)

본 문서는 어그로필터(AggroFilter)의 AI 분석에 적용된 실시간 동적 과금(Dynamic Billing) 로직과 계산 플로우를 정의합니다.

## 1. 과금 철학 (Billing Philosophy)
- **투명성**: AI API의 실제 사용량(토큰 및 검색)에 기반하여 과금합니다.
- **수익성**: API 원가 대비 약 30배(베이스 10배 * 마진 3배)의 수익 구조를 유지합니다.
- **합리성**: 검색 비용은 실제 효율과 무료 티어를 고려하여 정량제로 저렴하게 책정합니다.

## 2. 상세 계산 공식 (Calculation Formula)

### [A] 원가 산출 (Raw Cost)
모든 AI 모델과 도구의 사용량을 '토큰' 단위로 환산하여 합산합니다.

1.  **AI 토큰 (AI Tokens)**: 
    - **Speed Track**: `GPT-4o-mini` 모델이 썸네일과 자막 요약 시 사용하는 토큰
    - **Full Track**: `Gemini 1.5 Flash` 모델이 정밀 분석 및 채점 시 사용하는 토큰
    - **계산식**: `(GPT-4o-mini 토큰 + Gemini 1.5 Flash 토큰) / 1,000`

2.  **구글 뉴스 검색 (Google Search Grounding)**: 
    - **사용 모델**: `Gemini 1.5 Flash` (실시간 팩트 체크 도구 활성화)
    - **원가 산출**: 실제 효율을 고려한 **정량제(5C)** 적용
    - **계산식**: `구글 검색 횟수 * 5C`

3.  **합계 원가 (Total Raw Cost)**: 
    - **계산식**: `[1번 항목] + [2번 항목]`

### [B] 최종 청구 (Final Charge)
운영 마진을 적용하여 유저의 지갑에서 최종 차감합니다.
- **최종 금액**: `합계 원가 * 3 (마진 Multiplier)`
- **최소 과금**: 시스템 에러 및 최소 운영비 보전을 위해 분석당 최소 **10C**를 보장합니다.

## 3. 데이터 흐름 (Data Flow)

1.  **분석 수행**: `lib/openai-speed.ts`와 `lib/gemini.ts`가 각각 호출되어 분석을 완료합니다.
2.  **데이터 반환**: 각 모델이 사용한 정확한 **토큰 수**와 **검색 쿼리 목록**이 호출처(`route.ts`)로 전달됩니다.
3.  **비용 합산**: `app/api/analysis/request/route.ts`에서 위 공식[A]에 따라 원가를 계산합니다.
4.  **최종 결제**: Merlin Hub 백엔드로 `dynamic` 요청을 보내 실시간 잔액을 차감합니다.
5.  **증거 기록**: `t_analyses` 테이블의 `f_tokens_speed`, `f_tokens_full`, `f_grounding_count` 컬럼에 실사용 데이터를 영구 저장합니다.

## 4. 과금 예시 (Billing Examples)
- **일반 영상 (검색 없음, 1.2만 토큰)**: `(12C + 0C) * 3 = 36C` 차감
- **뉴스/시사 영상 (검색 1회, 1.5만 토큰)**: `(15C + 5C) * 3 = 60C` 차감
