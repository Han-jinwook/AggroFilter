# 🪙 어그로필터 정밀 동적 과금 시스템 가이드 (v1.1)
> **작성 일시**: 2026-05-11 21:45 (KST)

본 문서는 어그로필터(AggroFilter)의 AI 분석에 적용된 실시간 동적 과금(Dynamic Billing) 로직과 계산 플로우를 정의합니다.

## 1. 과금 철학 (Billing Philosophy)
- **투명성**: AI API의 실제 사용량(토큰 및 검색)에 기반하여 과금합니다.
- **수익성**: API 원가 대비 약 30배(베이스 10배 * 마진 3배)의 수익 구조를 유지합니다.
- **안정성**: 1,000토큰당 1C라는 직관적인 환율을 적용합니다.

## 2. 상세 계산 공식 (Calculation Formula)

### [A] 원가 산출 (Raw Cost)
모든 AI 모델과 도구의 사용량을 '토큰' 단위로 환산하여 합산합니다.
1.  **AI 토큰**: `(GPT-4o-mini 토큰 + Gemini 1.5 Flash 토큰) / 1,000`
2.  **구글 검색**: `구글 검색 횟수 * 50C` (회당 약 50원 원가 기준)
3.  **합계 원가**: `1 + 2`

### [B] 최종 청구 (Final Charge)
운영 마진을 적용하여 유저의 지갑에서 최종 차감합니다.
- **최종 금액**: `합계 원가 * 3 (마진Multiplier)`
- **최소 과금**: 에러 방지를 위해 분석당 최소 **10C**를 보장합니다.

## 3. 데이터 흐름 (Data Flow)

1.  **AI 실행**: 
    - `lib/openai-speed.ts` (GPT)와 `lib/gemini.ts` (Gemini)에서 분석 수행 후 `usage` 데이터를 호출처로 반환합니다.
2.  **데이터 합산**: 
    - `app/api/analysis/request/route.ts`에서 각 모델의 토큰과 검색 쿼리 개수를 취합합니다.
3.  **결제 처리**: 
    - `chargeDynamic` 함수를 통해 Merlin Hub 백엔드로 결제를 요청합니다.
4.  **잔액 차감**: 
    - 허브의 `WalletService`가 유저의 잔액을 실시간으로 업데이트하고 트랜잭션을 생성합니다.
5.  **기록 저장**: 
    - `t_analyses` 테이블의 전용 컬럼(`f_tokens_speed`, `f_tokens_full`, `f_grounding_count`)에 과금 근거 데이터를 영구 저장합니다.

## 4. 예시 (Example)
- **일반 영상 (검색 없음)**: 10,000토큰 사용 시 -> `(10C + 0C) * 3 = 30C`
- **정밀 분석 (검색 1회)**: 10,000토큰 + 검색 1회 -> `(10C + 50C) * 3 = 180C`
