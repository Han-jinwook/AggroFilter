# 📜 Merlin SDK 공식 5대 모듈 분류 체계

본 문서는 Merlin Hub SDK의 5가지 핵심 모듈 분류 체계를 정의합니다. 인앱 푸시 및 알림 종/뱃지 관련 기능이 전면 폐지됨에 따라 기존 6대 모듈에서 5대 모듈로 축소 개편되었습니다.

---

## 📂 5대 모듈 분류 요약

| 카테고리 | SDK 폴더명 | 주요 부품 (Component / Hook) | 역할 |
| :--- | :--- | :--- | :--- |
| **1. 인증 (Auth)** | `Auth` | `HubAuthModal`, `useHubAuth` | 6자리 OTP 기반 통합 로그인 및 회원가입 |
| **2. 세션/프로필 (Session)** | `Session` | `HubProfileWidget`, `useHubSession`, `HubAvatar` | 닉네임, 아바타 설정, 이메일 수신 동의 설정 및 현재 로그인 세션 감지 |
| **3. 지갑 (Wallet)** | `Wallet` | `HubPurchaseWidget`, `useHubPayment` | KCP 결제 충전, 잔액 조회 및 코인 이용 내역 |
| **4. 추천 (Referral)** | `Referral` | `HubReferralWidget`, `useHubReferral` | 초대 코드 발급 및 친구 초대(500C 보상) 연동 |
| **5. 코어 (CoreLogic)** | `CoreLogic` | `HubProvider`, `HubNotifier`, `hubFetch` | API 호출 규격, 로컬 스토리지 키 제어, 공통 테마 및 토스트 알림 |

---

## 🔍 상세 모듈 정의 및 가이드

### 🔑 1. 인증 (Auth)
* **목적**: Merlin Hub 생태계 전반의 공통 계정(이메일) 인증 및 가입을 제어합니다.
* **주요 기능**:
  * 6자리 OTP(One-Time Password) 요청 및 검증.
  * 로그인/가입 성공 시 공통 JWT 발급 및 브라우저 세션 적재.
* **주의**: 인증 세션이 성공적으로 처리된 이후에만 다른 지갑이나 추천 기능이 활성화됩니다.

### 👤 2. 세션 및 프로필 (Session)
* **목적**: 로그인된 유저의 프로필(닉네임, 아바타 등) 정보를 UI에 표출하고 갱신하며, 이메일 알림 수신 설정 변경을 처리합니다.
* **주요 기능**:
  * GUEST(비회원) 및 MEMBER(회원) 상태 구분 렌더링.
  * 프로필 변경 시 실시간 세션 캐시 갱신 및 헤더 위젯 동기화.
  * `useHub`의 `updateNotificationSettings` 헬퍼 메소드를 통해 이메일 수신 동의/해제 토글을 처리합니다.

### 💰 3. 지갑 (Wallet)
* **목적**: 사용자의 자산(코인) 충전, 잔액 감시 및 사용 내역 관리를 통합합니다.
* **주요 기능**:
  * KCP 표준 결제창 호출 연동 (`useHubPayment` - 신용카드 지원 / 휴대폰결제 심사 중).
  * 실시간 잔액 표시 및 충전 상품 카드 렌더링.
  * 이용 내역 페이징 조회.

### 🤝 4. 추천 및 보상 (Referral)
* **목적**: 생태계 활성화를 위한 추천인 코드 배포 및 피초대자 실사용에 따른 리워드를 제어합니다.
* **주요 기능**:
  * 나의 추천 코드 복사 및 링크 생성 위젯.
  * 가입 시 초대 코드 전달.
  * 가계정 방지 안전장치 탑재 (피초대자가 최초 코인 소모 시점에만 초대자에게 500C 보상 지급).

### ⚙️ 5. 코어 (CoreLogic)
* **목적**: 모든 모듈이 공통으로 사용하는 유틸리티, API Fetcher, 상태 공급자(Provider) 및 화면용 토스트 알림을 정의합니다.
* **주요 기능**:
  * `HubProvider`를 통한 앱 아이디(`appId`) 주입 및 글로벌 세션 상태 공유.
  * API 요청 시 CORS 및 JWT 헤더 자동 삽입 (`hubFetch`).
  * 화면 구석에 피드백을 노출하는 시스템 토스트 알림 컴포넌트(`HubNotifier`, `showToast`) 제공.
