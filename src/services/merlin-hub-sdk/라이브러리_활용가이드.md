# 📖 멀린 SDK 라이브러리 활용 가이드 (Index & Setup)
Version: v1.5.4
Last Updated: 2026-06-02

본 문서는 Merlin Hub SDK의 기본 설정(Setup) 절차와 각 도메인별 세부 연동 가이드 문서의 인덱스를 제공합니다.

---

## 🛠️ 0. 기본 설정 (Setup)

1. **에셋 복사**: `허브_라이브러리/assets/`의 파일들을 개별 앱의 `public/hub_assets/` 폴더로 복사합니다.
2. **Core 라이브러리 연동**: `허브_라이브러리` 내의 모든 폴더(Auth, Session, Wallet 등)와 파일을 앱의 `src/services/merlin-hub-sdk/` 경로에 배치합니다.

### 2단계: 에셋(이미지) 수동 배치 (필수)
부품들이 사용하는 공식 이미지들을 브라우저가 인식할 수 있도록 배치합니다.
1. 개별 앱의 `public/` 폴더 내에 `hub_assets/` 폴더를 생성합니다.
2. `src/services/merlin-hub-sdk/assets/` 내의 모든 파일을 `public/hub_assets/`로 복사합니다.
   - 예: `guest_paw.png` → `public/hub_assets/guest_paw.png`

### 3단계: 통합 배선 및 레이아웃 설정
1. **입구 이원화 (필수)**: 
   - **`@/src/services/merlin-hub-sdk`**: API Route, Server Component용 (Pure TS)
   - **`@/src/services/merlin-hub-sdk/react`**: UI, Hooks용 ('use client' 선언됨)
2. **알림 연동**: `app/layout.tsx` (또는 최상위 레이아웃)에 `<HubNotifier />`를 배치하여 시스템 전체 알림을 활성화합니다.

## 🎨 2. UX/UI 템플릿 표준 규격

- **사이드 윙 배치 공식 및 간격 (CSS 변수 기준)**:
  - **위치 공식**: 
    - `left: calc(50% + (var(--app-max-width) * 0.5 + var(--app-wing-gutter)))`
    - `top: calc(var(--app-header-height, 80px) + 16px)` (각 앱마다 헤더 높이가 다르므로 `--app-header-height` 변수를 기반으로 동적 16px 여백 확보)
  - **관련 규격**:
    - `--app-max-width`: `800px` (본문 최대 가로폭)
    - `--app-header-height`: 각 앱에 정의된 헤더 높이 픽셀 (예: 어그로필터 80px, 타 앱 72px 등 가변 적용)
    - `--app-wing-gutter`: `6px` (본문 박스 끝선 ~ 사이드 윙 사이의 마진)
    - `--app-wing-width`: `160px` (사이드 윙 가로폭)
  - **시각적 실질 간격**: 본문 박스 내부에 걸린 패딩(`px-4` = 16px)과 윙 간격(`6px`)이 합산되어, 화면 상에 렌더링된 본문 카드 우측선과 공유 카드 좌측선 사이의 실제 시각적 여백은 **`22px`**가 됩니다.
- **반응형 하단 광고 전환**: 화면 폭이 좁아져 우측 광고가 사라질 때는, 하단 앵커(Sticky Bottom) 등으로 대체합니다.

### 2.2 전 페이지 레이아웃 가로폭 및 패딩 표준 규격
패밀리 앱 전체의 통일감 있는 고품격 UX 구현을 위해 아래 **가로폭 수치(Pixel)**를 정확히 준수하여 모든 페이지와 카드 컴포넌트를 설계해야 합니다.

* **헤더 및 모든 페이지 최대 가로폭 (공통)**: **`800px`** (`max-w-[800px]` 또는 `max-w-[var(--app-max-width)]`)
  * 헤더(AppHeader)와 모든 페이지 메인 콘텐츠 영역의 가로폭은 **무조건 `800px`로 동일하게 제한**하여 좌우 정렬을 완전히 일치시킵니다.
  * **컨테이너 좌우 패딩 통일 (필수)**: 헤더 내부 컨테이너와 모든 페이지 메인 콘텐츠 컨테이너의 좌우 패딩은 모바일 및 데스크톱 환경 전체에서 동일하게 **`px-4`** (16px)로 완전히 일치시킵니다. 본페이지에 `lg:px-8`과 같은 차등 패딩을 선언할 경우, 데스크톱 환경에서 본문 영역만 안쪽으로 좁아져 정렬선이 깨지므로 절대 이를 금지합니다.
  * **헤더와 본문 세로 간격 표준화 (필수)**: 가변적인 헤더 높이에 대응하여 헤더 하단 경계선과 본문 카드 시작점 사이의 간격은 항상 **`16px`**로 통일합니다. 본문 `main` 컨테이너 상단에 `pt-4` (16px) 혹은 헤더 고정식의 경우 `pt-[calc(var(--app-header-height)+16px)]`를 주어 여백을 칼같이 일치시켜야 합니다.
* **레이아웃 그리드 표준 규격 (2컬럼 고정)**:
  * 본페이지는 무조건 **2컬럼 그리드** (`grid grid-cols-1 lg:grid-cols-2 gap-8`)로만 구성하여 메인 카드들이 각각 **`384px`**의 가로폭을 정상적으로 확보하도록 합니다.
  * 단일 컬럼 렌더링 시(로그아웃 등)에는 최대 가로폭 **`576px`** (`max-w-xl`)로 제한해 중앙 정렬합니다.
* **공유 카드 (`HubShareSquare`) 배치 절대 규칙**:
  * **본페이지 내부(800px 그리드 안) 배치 절대 금지**: 공유 카드는 본페이지 내부에 절대 임베딩하지 않습니다.
  * **우측 외부 고정 배치 (필수)**: 공유 카드는 항상 본페이지 레이아웃 외부의 **우측 영역(사이드 윙)**에 독립적으로 고정 플로팅하여 배치해야 합니다. (이를 통해 메인 카드 영역의 가로폭을 완벽히 보호하고, 정렬 일관성을 보장합니다.)
