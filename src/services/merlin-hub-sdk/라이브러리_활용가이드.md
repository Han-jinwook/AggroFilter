# 📖 멀린 SDK 라이브러리 활용 가이드
Version: v1.2.2
Last Updated: 2026-05-19

본 문서는 Merlin Hub SDK의 기술적인 사용 방법과 모든 모듈별 예제 코드를 관리합니다.

---

## 🛠️ 0. 기본 설정 (Setup)

1. **에셋 복사**: `허브_라이브러리/assets/`의 파일들을 개별 앱의 `public/hub_assets/` 폴더로 복사합니다.
2. **Core 라이브러리 연동**: `허브_라이브러리` 내의 모든 폴더(Auth, Session, Wallet 등)와 파일을 앱의 `src/services/merlin-hub-sdk/` 경로에 배치합니다.

### 2단계: 에셋(이미지) 수동 배치 (필수)
부품들이 사용하는 공식 이미지들을 브라우저가 인식할 수 있도록 배치합니다.
1. 개별 앱의 `public/` 폴더 내에 `hub_assets/` 폴더를 생성합니다.
2. `src/services/merlin-hub-sdk/assets/` 내의 모든 파일을 `public/hub_assets/`로 복사합니다.
   - 예: `guest_paw.png` → `public/hub_assets/guest_paw.png`

### 3단계: 통합 배선 및 레이아웃 설정
1. **입구 이원화 (필수)**: 
   - **`@/src/services/merlin-hub-sdk`**: API Route, Server Component용 (Pure TS)
   - **`@/src/services/merlin-hub-sdk/react`**: UI, Hooks용 ('use client' 선언됨)
2. **알림 연동**: `app/layout.tsx` (또는 최상위 레이아웃)에 `<HubNotifier />`를 배치하여 시스템 전체 알림을 활성화합니다.

### 🏁 연동 완료 체크리스트 (v1.2.0+)
- [ ] **구조**: 도메인별 폴더(Auth, Wallet, Session 등)가 정상적으로 복사되었는가?
- [ ] **UUID 통신**: JWT가 제공하는 `userId`가 DB의 `id`와 완벽히 일치하는지 확인했는가?
- [ ] **브랜딩**: 유저 노출 UI에서 "패밀리", "멀린", "허브" 등의 내부 용어가 제거되었는가?
- [ ] **에셋**: `public/hub_assets/guest_paw.png` 및 앱 전용 로고가 설정되었는가?
- [ ] **세션**: `useHubSession`에서 `profile.success`를 검사하는가?

> **📝 [메모] 타 앱 교차 진입 시 (자동 로그인) 가입 축하 코인 정책**
> 허브 생태계 내 다른 앱으로 넘어갈 경우, 해당 앱에서 '최초 기능 사용(코인 소모 액션)'을 할 때 이를 가입으로 간주하여 "해당 앱 가입 축하 500C"를 다시 지급해야 합니다.
> 단, 코인을 사용하지 않는 앱(예: 썬드리머)의 경우 축하 500C 지급 로직은 없습니다. (각 개별 앱의 서비스 로직에서 이 최초 액션 트리거를 구현해야 합니다.)

## 💳 3. 결제 및 코인 (Payment Suite)
- **부품**: `useHubPayment` (Core), `HubPaymentTrigger` (Custom), `HubPurchaseWidget` (Custom)
- **용도**: KCP 결제창 호출, 지갑 잔액 조회 및 공통 결제/이동 내역 UI 페이지 제공.

### [Custom] `HubPurchaseWidget` Component (v1.0.0)
유저의 실시간 잔액 표시, 충전/이용내역 탭 전환, 1,000C / 5,000C / 10,000C 상품 카드 렌더링, 결제 수단 선택(신용카드/휴대폰), 결제창 브릿지 연동, 이용 내역 조회 및 페이징 처리까지 모든 결제 관련 UI와 로직을 내장한 올인원 위젯입니다.

```tsx
import { HubPurchaseWidget } from '@/src/services/merlin-hub-sdk/react';

// Next.js Page 컴포넌트 내에서 간편 호출
<HubPurchaseWidget 
  appName="어그로필터" 
  redirectUrl="/" 
/>
```

### [Core] `useHubPayment` Hook (v1.3.0)
KCP 결제 준비 및 승인 통신을 중개하는 Hook입니다. `'card'`(신용카드) 결제를 공식 지원하며, `'phone'`(휴대폰결제)은 현재 심사 중입니다. (계좌이체는 정부 권고 공문에 따라 제거되었습니다.)


### 📊 동적 과금 한도 규격화 정책 (Pricing Cap Policy)
각 개별 앱은 Supabase `family_apps` 테이블의 `client_secret` 필드 내 JSON 형태로 동적 과금 및 마진 배수를 제어할 수 있습니다.
- **`margin_multiplier`**: 동적 요금 계산 시 적용할 곱연산 마진 배수
- **`token_to_coin_rate`**: 토큰 사용량을 코인으로 환산하는 비율 (기본값: 1000토큰 = 1C)
- **`min_price`**: 과금 시 최소로 보장할 최저 코인 금액 (기본값: 10C)
- **`max_price`**: 과금 시 최대로 부과할 최대 코인 금액 한도 (지정하지 않을 시 제한 없음)

*예시 (어그로필터):*
```json
{
  "margin_multiplier": 3,
  "token_to_coin_rate": 1000,
  "min_price": 10,
  "max_price": 100
}
```
위 설정에 따라, 아무리 큰 비디오(예: 70,000토큰 이상)를 분석하더라도 **최대 100C**를 넘지 않도록 제한 한도가 자동으로 안전하게 적용됩니다.

## 👥 6. 개별 앱 유저 연동 전략 (User Integration)

개별 앱(Individual App)은 고유의 사용자 테이블을 유지하며, 허브의 공식 식별자를 연동 키로 사용합니다.

### 1단계: DB 스키마 설계 표준 (Standard)
신규 패밀리 앱 개발 시 아래와 같은 네이밍 표준을 권장합니다.
- **테이블 명**: **`users`**
- **연동 컬럼**: **`id`** (UUID) — 허브의 `family_users.id` 값을 그대로 저장합니다.
- **참고**: 어그로필터와 같이 이미 구축된 앱은 `t_users`, `f_id` 등 고유의 네이밍을 사용할 수 있으나, 신규 연동 시에는 위 표준을 따릅니다.

### 2단계: 세션 동기화 (Upsert 패턴)
로그인 성공 시 반환되는 `user.id`를 사용하여 로컬 유저 레코드를 생성하거나 업데이트합니다.

```typescript
// 예시: 허브 로그인 완료 후 로컬 DB(users)와 동기화
const { user, isLoggedIn } = useHub();

useEffect(() => {
  if (isLoggedIn && user) {
    // 로컬 API 호출: 
    // 1. users 테이블에서 id === user.id 인 레코드가 있는지 확인
    // 2. 없으면 신규 생성(Insert), 있으면 최신 정보 업데이트(Update)
    fetch('/api/auth/sync-user', {
      method: 'POST',
      body: JSON.stringify({ 
        hub_id: user.id, // 허브의 family_users.id
        email: user.email 
      })
    });
  }
}, [isLoggedIn, user]);
```

### 3단계: 데이터 관리 책임 분할
- **허브(Hub)**: `family_users` (계정), `family_wallet_balances` (통합 잔액).
- **로컬(Local)**: `users` (앱 전용 설정, 포인트, 활동 로그 등).

## 🎁 7. 초대 시스템 (Referral System) 연동

친구 초대 시 **초대자(Inviter)**와 **가입자(Invitee)** 모두에게 보상을 지급하는 시스템입니다.

### 1단계: 초대 링크 생성
초대자의 코드를 URL 파라미터에 포함하여 홍보 링크를 생성합니다.
- **형식**: `https://your-app.com?ref=INVITE_CODE`
- **SDK 위젯**: `HubReferralWidget`을 배치하면 유저가 자신의 코드를 즉시 확인하고 복사할 수 있습니다.

### 2단계: 가입 시 초대 코드 자동 인식
앱의 진입점(Landing 또는 Login 페이지)에서 URL의 `ref` 파라미터를 저장했다가 가입 시 허브로 전달합니다.

```typescript
// 예시: 초대 코드를 로컬 스토리지에 임시 저장
useEffect(() => {
  const urlParams = new URLSearchParams(window.location.search);
  const refCode = urlParams.get('ref');
  if (refCode) {
    localStorage.setItem('pending_invite_code', refCode);
  }
}, []);

// 회원가입/로그인 시 허브 SDK에 전달
const { login } = useHub();
const pendingCode = localStorage.getItem('pending_invite_code');

const handleJoin = (email, otp) => {
  login(email, otp, pendingCode); // 세 번째 인자로 초대 코드 전달
};
```

### 3단계: 보상 정책 (Hub 기본값)
- **초대자(Inviter)**: 친구 가입 후 최초 기능 소모(가불 정산 또는 분석) 완료 시 **500C** 즉시 지급
- **가입자(Invitee)**: 추천 가입에 따른 별도 추가 보상은 없으나, 모든 가입자는 최초 기능 사용 시 기본 웰컴 보상 **500C** 지급 (초대자 보상은 피초대자의 실제 사용이 일어나기 전까지 PENDING으로 안전하게 보호되어 가계정 어뷰징을 자동 방지합니다.)

## 🎁 8. 비회원 선(先) 체험 가불 정산 및 데이터 이관 스펙

유저에게 가입이나 결제를 요구하기 전에 서비스 핵심 기능을 1회 맛보게 하고, 로그인 가입 시 지급되는 웰컴 보상(`500C`)에서 체험 비용을 사후 차감함과 동시에 해당 활동 이력을 진짜 유저 계정(UUID)으로 이관하는 표준 스펙입니다.

- **스토리지 키 표준**:
  - `pending_usage_fee`: 맛보기 시 발생한 가불 코인 금액 (예: `34`)
  - `pending_video_id`: 맛보기로 생성/분석된 로컬 엔티티 고유 ID
- **인증 연동**: `verifyOTP` 호출 시 `pendingUsageFee`와 `pendingVideoId` 인자를 전달하면 허브 백엔드가 원자적 결제를 대행합니다.
- **데이터 이관**: 가입 성공 직후 개별앱 내 `/api/analysis/link-guest`와 같은 엔드포인트를 호출하여 해당 비디오/기록의 `f_user_id`를 임시 ID(`trial_...`)에서 정식 유저 ID(진짜 UUID)로 변경합니다.

> 📘 **상세 스펙 가이드**: 상세한 데이터 흐름 시퀀스 다이어그램 및 개별앱 연동 코드는 [비회원 선체험 가불정산 및 데이터 이관 가이드](../docs/비회원_선체험_가불정산_및_데이터_이관_가이드.md)를 참고해 주시기 바랍니다.

## 🕵️‍♂️ AI 개발자 필독 사항
- 작업 전 반드시 `라이브러리_대장.md`에서 최신 버전을 확인하십시오.
- **용어 표준 준수**: 모든 문서와 코드에서 **Inviter(초대자) / Invitee(가입자)** 명칭을 엄격히 준수하십시오.

- **Source-First**: 모든 수정 사항은 원본(`MerlinFamilyOS/허브_라이브러리`)에 먼저 적용합니다.
