# 🧩 Profile, Settings & Session Integration Guide

본 문서는 Merlin Hub SDK의 프로필, 설정 카드 및 세션 관리 모듈에 대한 연동 명세입니다.

---

## 1. 주요 모듈 & Hooks
- **부품 Hooks**: `useHubSession` (Core), `useHub` (Core Logic)
- **부품 Components**: 
  - `HubProfileWidget`, `HubAvatar` (헤더 아바타 위젯)
  - `HubProfileCard` (프로필/닉네임/사진 수정 카드, `hideCoinSection` 속성 지원)
  - `HubNotificationCard` (단일 스마트 알림 동의 카드)
  - `HubLogoutCard` (계정 로그아웃 카드)

---

## 2. 설정 페이지 (/p-settings) 조립 표준 & 레이아웃 규격

개별 패밀리 앱들은 팝업 모달 대신 독립된 페이지(`/p-settings`)를 제공하며, 허브 부품 카드가 화면 크기에 맞춰 조화롭게 배치되도록 아래 **레이아웃 표준 규격**을 반드시 준수하여 조립해야 합니다.

### 📐 레이아웃 표준 규격 (Desktop/Mobile)
- **컨테이너 가로폭**: 메인 페이지 래퍼는 최대 가로폭 **`max-w-4xl (896px)`**로 엄격히 제한합니다. 가로폭 제한이 없으면 카드가 가로로 비정상적으로 늘어나(Stretched) 디자인 톤앤매너가 훼손됩니다.
- **그리드 구성**: 데스크톱 환경(lg 이상)에서는 **2컬럼 그리드 (`grid-cols-1 lg:grid-cols-2 gap-8`)** 구성을 표준으로 합니다.
  - **좌측 컬럼 (Left Column)**: 초대 실적 현황 (`HubHistoryList`) 및 초대 혜택 정보 등 추천인 관련 정보 배치.
  - **우측 컬럼 (Right Column)**: 프로필 정보 (`HubProfileCard`), 알림 설정 (`HubNotificationCard`), 로그아웃 (`HubLogoutCard`) 배치.
  - **로그아웃 상태**: 비로그인 상태이거나 초대 실적이 없을 경우, 우측 컬럼(프로필 영역)을 가로폭 **`max-w-xl (36rem / 576px)`** 단위의 단일 컬럼으로 중앙 정렬하여 렌더링합니다.

### 💻 표준 조립 예시 (Next.js / Tailwind CSS 기준)
```tsx
import { 
  HubProfileCard, 
  HubNotificationCard, 
  HubLogoutCard,
  HubHistoryList,
  useHub,
  useHubReferral
} from '@/src/services/merlin-hub-sdk/react';
import { useRouter } from 'next/navigation';
import { useState, useEffect } from 'react';

export default function SettingsPage() {
  const router = useRouter();
  const { isLoggedIn } = useHub();
  const { getReferralHistory } = useHubReferral();
  const [referralHistory, setReferralHistory] = useState<any[]>([]);
  const [isHistoryLoading, setIsHistoryLoading] = useState(false);
  const [smartNotification, setSmartNotification] = useState(true);

  useEffect(() => {
    if (isLoggedIn) {
      setIsHistoryLoading(true);
      getReferralHistory()
        .then(res => { if (res) setReferralHistory(res); })
        .catch(err => console.error(err))
        .finally(() => setIsHistoryLoading(false));
    }
  }, [isLoggedIn]);

  return (
    <div className="min-h-screen bg-slate-50">
      {/* 개별 앱 헤더 */}
      <AppHeader />
      
      {/* 1. 메인 컨테이너: 최대 가로폭 896px 제한 (max-w-4xl) */}
      <main className="container mx-auto px-4 lg:px-8 py-8 max-w-4xl">
        <div className="grid grid-cols-1 lg:grid-cols-2 gap-8">
          
          {/* [좌측 컬럼] 초대 실적 현황 (로그인 시에만 노출) */}
          {isLoggedIn && (
            <div className="w-full flex flex-col gap-6">
              <HubHistoryList history={referralHistory} isLoading={isHistoryLoading} />
            </div>
          )}

          {/* [우측 컬럼] 프로필 정보 및 알림/로그아웃 설정 */}
          {/* 로그인하지 않은 경우 가로폭 max-w-xl(576px) 단일 컬럼 중앙 정렬 */}
          <div className={`w-full space-y-6 ${!isLoggedIn ? 'lg:col-span-2 max-w-xl mx-auto' : ''}`}>
            
            {/* 프로필 카드 */}
            {/* 왓잇앱 등 코인 기능이 없는 앱(무코인 앱)은 hideCoinSection={true} 속성을 주입하여 코인 영역을 숨깁니다. */}
            <HubProfileCard hideCoinSection={isCoinlessApp} />

            {/* 알림 설정 카드 (이메일 알림 기본 렌더링 권장) */}
            <HubNotificationCard />

            {/* 로그아웃 카드 */}
            <HubLogoutCard onLogout={() => router.push('/')} />
          </div>
          
        </div>
      </main>
    </div>
  );
}
```

> 🚨 **[중요] 불필요한 개별 헤더/타이틀 중복 배치 금지**
> `HubProfileCard` 및 `HubNotificationCard` 등 각 카드 블록 내부에는 자체적인 타이틀 영역("프로필 정보", "알림 설정" 등)이 이미 포함되어 있습니다.
> **따라서 카드 상단에 별도로 "설정", "프로필 및 알림" 같은 자체 헤더 텍스트나 부연설명글을 중복으로 하드코딩하지 마십시오. 화면이 지저분해지고 UX가 훼손됩니다.**
