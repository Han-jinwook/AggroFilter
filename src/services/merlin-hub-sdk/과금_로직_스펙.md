# 🪙 어그로필터 정밀 동적 과금 시스템 가이드

본 문서는 어그로필터(AggroFilter)의 AI 분석에 적용된 실시간 동적 과금(Dynamic Billing) 로직과 계산 플로우를 정의합니다.

## 1. 과금 철학 (Billing Philosophy)
- **투명성**: AI API의 실제 사용량(토큰 및 검색)에 기반하여 과금합니다.
- **수익성**: API 원가 대비 약 3배의 합리적인 마진을 유지합니다.
- **합리성**: 1C = 1원 기준으로, 대용량 분석도 부담 없는 가격으로 제공합니다.

## 2. 상세 계산 공식 (Calculation Formula)

### [A] 원가 산출 (Raw Cost)
모든 AI 모델 API 호출 시 사용된 실제 토큰 소모량을 '코인(C)' 단위로 환산하여 합산합니다.

1. **AI 토큰 (AI Tokens)**: 
   - **환율**: **10,000 토큰 = 1C**
   - **대상 모델**: `GPT-4o-mini` (Speed), `Gemini 2.5 Flash` (Full)
   - **계산식**: `(GPT 실제 소모 토큰 + Gemini 실제 소모 토큰) / 10,000`
   - *참고: Gemini의 실시간 검색(Grounding)을 사용하는 경우 검색 과정에서 발생한 토큰이 API 자체의 totalTokenCount에 실제 데이터로 포함되어 반환되므로, 인위적인 가중치 없이 이 실소모 토큰을 100% 신뢰하여 계산합니다.*

### [B] 최종 청구 (Final Charge)
운영 마진을 적용하여 유저의 지갑에서 최종 차감합니다.
- **최종 금액**: `원가 코인 * 3 (마진 Multiplier)`
- **최소 과금**: 분석당 최소 **10C**를 보장합니다.
- **최대 과금**: 분석당 최대 **100C**를 한도로 적용합니다.

## 3. 데이터 흐름 (Data Flow)

1.  **분석 수행**: 각 모델이 사용한 정확한 실제 토큰 수를 취합합니다.
2.  **사용량 전송**: 개별 앱은 소모한 실제 사용량 메트릭(`speedTokens`, `fullTokens`)을 SDK를 통해 허브 백엔드로 그대로 전송합니다. **개별 앱 내에서는 어떠한 과금 관련 환산 로직도 직접 실행하지 않습니다.**
3.  **비용 계산 및 결제**: Merlin Hub 백엔드에서 전송받은 실제 토큰 수에 기초하여 **10,000:1** 환율을 적용해 원가를 산출한 후, 마진(3배)을 곱하고 앱 정책 한도(최소 10C / 최대 100C)를 대입해 최종 차감 금액을 정산 및 결제 처리합니다.

## 4. 과금 예시 (Billing Examples)
- **가벼운 영상 (실소모 1.2만 토큰)**: `((12,000 / 10,000) * 3) = 3.6C` -> 최소 가격 보장 정책에 의해 **10C** 차감
- **뉴스/시사 영상 (검색 사용으로 실소모 6.5만 토큰)**: `((65,000 / 10,000) * 3) = 19.5C` -> 올림하여 **20C** 차감
- **대용량 긴 영상 (실소모 35만 토큰)**: `((350,000 / 10,000) * 3) = 105C` -> 최대 가격 캡 정책에 의해 **100C** 차감
