# 📱 멀린 패밀리 개별 앱 정의와 특징 (통합 규격서)

본 문서는 멀린 패밀리 OS 생태계 내 개별 서비스의 공식 명칭, 역할 정의, 비즈니스 모델(BM), 가입 활성화 트리거 조건, 세션 정산 주기, 연동 인터페이스 및 DB 스키마 표준 규격까지 하나로 통합한 **일원화 규칙서**입니다.

이 문서가 생성됨에 따라 기존의 `앱별_과금정책.md` 문서는 완전히 대체 및 폐기(삭제)되며, 패밀리 내 모든 앱 개발은 본 문서의 통합 기준을 준수해야 합니다.

## 1. 개별 앱 기본 정의 및 비즈니스 모델 (BM)

패밀리 앱 생태계에 소속된 모든 앱은 고유한 역할과 BM 정책을 갖습니다.

| 공식 표준 ID (영문) | 국문 명칭 (표시용) | 서비스 포지셔닝 & 앱의 기본 성격 | 비즈니스 모델 (BM) |
| :--- | :--- | :--- | :--- |
| **`AggroFilter`** | 어그로필터 | 유튜브 영상의 어그로 필터링 및 리터러시 교육 보조 | **[코인 과금 + 부분 광고]**<br>- 신규 영상 분석 시 AI 리소스 비례 코인 차감<br>- 단순 DB 캐싱 조회(무료) 시에는 부분 광고 노출 |
| **`Vaulter`** | 금고지기 | AI 기반 영수증, 계약서 및 기밀 문서 분석/보관 | **[순수 코인 과금]**<br>- 광고 노출 없음<br>- AI 분석 리소스 전량을 코인으로 정밀 차감 |
| **`SunDreamer`** | 썬드리머 | 오프라인 실물 제품 구매 회원을 위한 서포트 및 소통 공간 | **[코인 없음]**<br>- 코인 차감 및 충전 시스템 없음<br>- 오프라인 구매 고객 케어에 특화 |
| **`WhatEat`** | 뭐먹지? | 전국 초중고 학교 급식 평가 랭킹 및 AI 급식퀴즈 배틀 | **[무료 광고 기반 + 로컬 관리]**<br>- 인앱 결제 및 코인 시스템 없음<br>- 배너 광고 및 보상형 광고 적용 |

> 💡 **가이드라인**: 신규 패밀리 앱 추가 시에는 독단적으로 개발하지 않고, **반드시 사장님(Merlin)께 서비스 성격, BM, 세션 및 가입 활성화 조건을 요구하여 이 장에 등재한 후** 연동 개발에 착수해야 합니다.

## 2. 앱별 가입 및 최초 활성화 (보상) 조건

신규 회원이 가입하거나, 기존 회원이 다른 패밀리 앱으로 교차 진입 시 적용되는 **가입 확정(최초 활성화) 및 보상(웰컴 페이백 500C / 추천 리워드 200C)** 기준입니다.

### 🚨 [중요] 통합 보상 지급 원칙 (가입 즉시 지급)
1. **가입자 웰컴 보상 (500C)**:
   * **유료(코인) 앱 최초 가입(로그인) 시 생태계 전체에서 단 1회만** 즉시 지급됩니다.
   * **무료 앱 (`WhatEat`, `SunDreamer` 등) 가입 시**: 가입자 웰컴 보상은 **지급되지 않습니다**. (가입 이력은 추적되나, 무료 앱 유저의 혼란을 막기 위해 코인 발급 및 UI 노출 없음)
   * 만약 무료 앱에 먼저 가입한 유저가 이후 유료 앱(`AggroFilter` 등)으로 이동하여 최초 로그인을 완료(SSO)하면, 그 시점에 **1회 최초 500C**를 즉시 지급받게 됩니다.
2. **초대자 추천 보상 (200C)**:
   * 피초대자(가입자)가 **유료(코인) 앱의 초대 링크를 통해 가입(로그인)을 완료한 즉시 단 1회만** 초대자에게 200C가 지급됩니다. (별도 어뷰징 방어막 없음)
   * **무료 앱의 초대 링크로 가입한 경우**: 초대자 보상은 영구적으로 **지급되지 않습니다**.
   * 무료 앱 초대로 가입한 피초대자가 나중에 스스로 유료 앱에 로그인하여 자신의 웰컴 500C를 받더라도, 최초 가입 출처가 무료 앱이었으므로 초대자에게는 보상이 **지급되지 않습니다**.

### 1) 🔍 어그로필터 (`AggroFilter`)
* **상태**: 가입(로그인) 완료 즉시 100% 활성화. (기존 가입 대기 단계 폐지)
* **결과**: 가입 즉시 웰컴 페이백(500C) / 추천인 보상(200C)이 즉시 지급됩니다. (비로그인 무료체험 후 발생한 가불금은 500C 수령 직후 자동차감됩니다.)

### 2) 🔐 금고지기 (`Vaulter`)
* **상태**: 가입(로그인) 완료 즉시 100% 활성화. (기존 가입 대기 단계 폐지)
* **결과**: 가입 즉시 웰컴 페이백(500C) / 추천인 보상(200C)이 즉시 지급됩니다. (비로그인 무료체험 후 발생한 가불금은 500C 수령 직후 자동차감됩니다.)

### 3) 🏡 썬드리머 (`SunDreamer`)
* **활성화 트리거**: 유저가 썬드리머 웹 애플리케이션에 **최초 로그인 또는 방문을 완료한 즉시**.
* **결과**: 즉시 100% 가입 확정 및 활성화 처리됩니다. (단, 썬드리머 자체는 코인 연동 앱이 아니므로 가입 축하 페이백 및 추천인 코인은 지급되지 않습니다.)

### 4) 🍱 뭐먹지? (`WhatEat`)
* **활성화 트리거**: 로그인(허브 연동) 완료 후, 유저가 웹 서비스 내에 **누적 3분(180초) 이상 실제 체류 및 활동**을 달성하여 왓잇 로컬 Supabase DB에 `is_activated = true`가 영구 기록된 시점.
* **결과**: 왓잇 서비스 내 혜택 잠금 해제 및 회원 활성화 처리. (단, 왓잇 자체는 코인이 없는 무료 광고 기반 앱이므로 허브 중앙의 웰컴 코인 및 추천인 보상은 지급되지 않고, 로컬 활성화 상태만 자체 관리합니다.)

## 3. 앱별 세션(Session) 및 과금 정산 상세 정책

### 1) 어그로필터 (`AggroFilter`)
* **세션 정의**: **매 영상 1건 분석 = 1세션** (분석 완료 즉시 실시간 정산 및 차감 수행)
* **테스트 세션 (무료 체험)**: 최초 1개 영상 분석 성공 즉시 선체험 종료되며 가불금(`pending_usage_fee`)을 기록합니다.
* **상세 요금 산출 수식**:
  * AI 텍스트 토큰: 10,000 Tokens = 1C (GPT-4o-mini 및 Gemini 2.5 Flash 공통 요율)
  * 구글 뉴스 검색(그라운딩): 검색 1회 = 5C 고정 요금 적용
  * **최종 청구 코인** = `Math.min(100, Math.max(10, Math.ceil((텍스트 토큰 원가 + 구글 검색 원가) * 3)))` (3배 운영 마진 적용, 최소 10C / 최대 100C 한도)

### 2) 금고지기 (`Vaulter`)
* **세션 정의**: **로그인 시점부터 매 30분 단위 고정 블록 (Time Window)**
* **정산 주기**:
  * 로그인 시점부터 30분마다 Netlify Blobs에 누적된 실사용 메트릭(GPT-4o-mini 토큰 등)을 읽어 정산 후 잔액을 차감합니다. (활동이 없다면 0C 정산)
  * 31분부터 새로운 30분 블록을 시작하여 동일하게 누적하고, 로그아웃 시에는 즉시 남은 자원을 1회 정산합니다.
* **테스트 세션 (무료 체험)**: 최초 1회 OCR 및 문서 분석 실행 성공 즉시 선체험이 종료되며 가불금을 기록합니다.
* **상세 요금 산출 수식**:
  * 원가 환산: 10,000 Tokens = 1C (`gpt-4o-mini` DB 요율 적용)
  * **최종 청구 코인** = `Math.max(1, Math.ceil(세션 누적 토큰 원가 * 10))` (10배 마진 적용, 최소 1C 한도)
  * 임시 보존 스토리지: 유저 조작 방지를 위해 사용 메트릭을 Netlify Blobs에 실시간 누적한 후 정산 시점에 1회 호출 후 클리어 처리합니다.

### 3) 썬드리머 (`SunDreamer`)
* **세션 정의**: **세션 및 정산 없음**
* **정산 주기**: 과금 및 결제 로직이 존재하지 않으므로, 정산 처리가 일어나지 않습니다.

### 4) 뭐먹지? (`WhatEat`)
* **세션 정의**: **세션 및 정산 없음**
* **정산 주기**: 결제 및 허브 코인 과금 로직이 존재하지 않으므로, 정산 처리가 일어나지 않습니다. (왓잇 로컬 DB를 통한 3분 체류 활성화 상태만 독립 관리)

## 4. 개별 앱 연동 표준 인터페이스 스펙

개별 앱이 SDK를 통해 허브 백엔드로 전달해야 하는 데이터의 공통 표준 규격입니다. 개별 앱 자체에서는 어떠한 비용 계산도 하지 않고, 자원이 소모된 실지 메트릭만 실어 보냅니다.

```typescript
interface ChargeDynamicParams {
  userId: string;         // 유저 UUID (family_users.id)
  videoId: string;        // 분석 대상 고유 리소스 식별자 (예: 유튜브 비디오 ID, 세션 ID 등)
  usageMetrics: {         // API 호출로 소모된 실제 토큰량 및 기능 사용 횟수
    gpt4oMiniTokens?: number;      // GPT-4o-mini 실제 소모 토큰 수
    gemini25FlashTokens?: number;  // Gemini 2.5 Flash 실제 소모 토큰 수
    googleSearchCount?: number;    // 구글 뉴스 검색(그라운딩) 실행 횟수
  };
  requestId: string;      // 이중 차감 방지용 고유 멱등성 키 (예: `fresh_${videoId}_${analysisId}`)
  displayText: string;    // 유저 이용내역(거래 히스토리)에 적을 메모 (예: `어그로필터 - 영상 분석 - [영상 제목]`)
}
```

## 6. 🚨 클라이언트 ID 송신 규정 (개발자 강령)

1. **표준 ID 사용 의무**: 모든 패밀리 앱의 클라이언트는 허브와 통신 시 위에 명시된 **표준 영문 ID (`AggroFilter`, `Vaulter`, `SunDreamer`)**를 대소문자까지 일치시켜 전달해야 합니다.
2. **에러 강제화**: 허브 백엔드는 `AGGRO_FILTER`, `LumiNote`, `sundreamer.app` 등 허용되지 않은 변종 ID 및 빈 ID가 송신될 경우, 기본값(멀린 허브 등)으로 처리하지 않고 **즉시 통신 실패(Error)로 차단**합니다.
3. **사용자 노출**: 유저 메일 발송, 잔액 및 충전 기록 화면 등 사용자에게 브랜드를 표기하는 텍스트에는 표준 ID가 아닌 **한글 앱 명칭 (`어그로필터`, `금고지기`, `썬드리머`)**이 다이렉트로 매핑되어 표시됩니다.
