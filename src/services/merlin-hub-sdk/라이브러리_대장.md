## 🔄 허브 인프라 변경 및 공지 (Hub Changelog)

- **2026-06-05**: **SDK v2.0.0 배포 / 인앱 푸시 완전 제거 및 이메일링 단독 유지 롤백**.
  - **인앱 푸시 롤백**: 알림 종 뱃지, 실시간 카운트 폴링 로직(`unreadCount`), 반투명 드롭다운 팝업(`HubNotificationDropdown`) 및 DB `family_notifications` 연동을 완전히 제거하여 단순화했습니다.
  - **설정 단순화**: 설정 카드(`HubProfileCards`) 내 알림 수신 동의를 '푸시' 항목 제거 후 오직 '이메일 알림 수신 동의' 하나로 축소 및 통일했습니다.
  - **이메일 브릿지 유지**: 백엔드`/api/notification/send` 및 어드민`/api/admin/notification/send` API는 유지되나, 인앱 저장 로직을 걷어내고 오직 Naver SMTP를 통한 프리미엄 이메일 단독 전송만 실행되도록 구조를 개편했습니다.
- **2026-06-03**: **SDK v1.6.0 배포 / 통합 알림 및 이메일링 연동 완료**.
  - **인앱 푸시 뱃지 및 리스트**: 헤더의 알림 종 클릭 시 안 읽은 알림 목록을 표시하는 Glassmorphism 기반 반투명 드롭다운 팝업(`HubNotificationDropdown`)을 도입하고, 30초마다 안 읽은 카운트를 자동 갱신하는 폴링 기능을 `HubProvider`에 탑재함.
  - **네이버 SMTP 연동**: 백엔드에 `nodemailer` 기반의 `email.util.ts`를 신설하여 이미지 첨부(`<img src="..." />`)가 포함된 HTML 프리미엄 딥블루 메일 템플릿 일괄 발송을 연동함.
  - **중요 공지사항 수신거부 우회 (isNotice)**: 장애 전파 및 점검 안내 등 긴급 공지 발송을 위해, 어드민에서 `isNotice: true` 옵션을 켜서 발송 시 푸시/이메일 수신 거부 설정을 한 유저에게도 무조건 메일링 및 인앱 알림이 도달하도록 백엔드 분기를 구축함.
  - **트리거 SDK 보강**: 개별 앱에서 알림을 바로 쏠 수 있는 API 메소드(`client.sendNotification`)와 React 훅 헬퍼(`useHubNotifier.triggerNotification`)를 정식화함.
- **2026-06-02**: **SDK v1.5.5 배포 / HubHistoryList 제목 '앱 초대 List' 변경**. 어그로필터 프로필 페이지 및 SDK의 초대 실적 카드 제목을 '앱 초대 List'로 변경하여 가독성을 개선하고 브랜딩 정체성을 명확히 함.
- **2026-06-02**: **SDK v1.5.4 배포 / 가변 헤더 대응 및 16px 세로 간격 표준화**. 각 서비스의 디자인 다양성(가변 헤더 높이)을 보장하면서도 본문 및 윙 카드와 헤더 사이의 간격을 일관되게 `16px`로 통일하기 위해, `--app-header-height` CSS 변수 활용 공식과 동적 top/margin 공식을 표준화하고 어그로필터에 즉시 적용함.
- **2026-06-01**: **SDK v1.5.3 배포 / HubHistoryList 리스트 압축 최적화**. 100개 이상의 초대 리스트 렌더링 시 세로 높이가 과도하게 비대해지는 문제를 해결하기 위해 각 리스트 항목을 2행 구조로 압축함. 1행에 닉네임과 이메일을 단독 노출(찢어짐 방지)하고, 2행에서 좌측에 가입일, 우측에 상태 뱃지를 정렬하여 세로 높이를 대폭 줄이고 가독성을 높임.
- **2026-06-01**: **SDK v1.5.2 배포 / 사이드 윙 표준화**. 본문 끝선과 우측 사이드 윙 간의 간격 불일치를 차단하기 위해, 배치 위치 결정 공식(`calc(50% + 400px + 6px)`)과 물리적 간격 수치(`--app-wing-gutter`: `6px`, 실질 시각 여백 `22px`)를 명확하게 규격화하여 문서에 공식 등재함.
- **2026-06-01**: **SDK v1.5.1 배포 / 좌우 패딩 표준화 및 정렬 일치**. 데스크톱 환경에서 본문 컨테이너만 더 좁아져 정렬선이 무너지는 현상을 해결하기 위해, 헤더와 본페이지 컨테이너의 좌우 패딩을 모바일/데스크톱 구분 없이 동일하게 `px-4`(16px)로 일치하도록 표준 규격화하고 어그로필터에 즉시 적용함.
- **2026-06-01**: **SDK v1.5.0 배포 / HubHistoryList 뱃지 하단 강제 배치**. 닉네임과 이메일이 길어질 경우 뱃지 영역이 공간을 차지해 닉네임이 줄바꿈되며 찢어지는 문제를 해결하기 위해, `HubHistoryList` 리스트 항목을 완전 수직 구조(`flex-col`)로 재설계하여 보상 지급 상태 뱃지를 항상 유저 정보 아래의 비어 있는 행(아랫줄)에 독립 렌더링하도록 수정함.
- **2026-06-01**: **SDK v1.4.9 배포 / HubHistoryList 뱃지 찢어짐 버그 해결**. 초대 실적 리스트에서 닉네임이 길어질 경우 우측의 보상 지급 뱃지(지급 완료 / 대기 중)가 비정상적으로 줄바꿈되어 찢어지는 문제를 `flex-wrap` 및 `whitespace-nowrap`을 적용하여 가로폭이 부족할 때 자연스럽게 아랫줄에 온전한 단어로 떨어지도록 개선함.
- **2026-06-01**: **SDK v1.4.8 배포 / 공유카드 우측 외부 고정 규격화**. 공유 카드(`HubShareSquare`)를 본페이지 내부에 임베딩하는 방식의 혼선을 차단하기 위해 본문 내 배치 절대 금지 및 항상 우측 외부 영역(사이드 윙)에 고정 플로팅하여 배치하도록 강제 규격화함.
