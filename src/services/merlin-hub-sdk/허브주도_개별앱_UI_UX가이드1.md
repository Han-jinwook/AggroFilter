# [제 1 부] 허브 전담 AI 통제 구역 (공통 인프라 & 레이아웃)

이 구역의 원칙들은 멀린 패밀리 생태계의 통일성을 위해 **허브 전담 AI가 직접 각 앱의 코드를 수정하여 배포(통제)**합니다. 개별 앱 AI는 이 구조를 임의로 훼손해서는 안 됩니다.

## 1. SSOT (Single Source of Truth) 원칙
- 유저 식별자(UUID)와 계정 정보, 지갑 정보는 오직 **허브 DB(`family_users`)만이 유일한 진실 공급원**입니다.
- 이메일이나 로컬 닉네임 등을 임의의 보조 수단으로 매핑하여 시스템을 우회하거나 어지럽히는 행위는 절대 금지됩니다.

## 2. 반응형 배너 분기 및 고정 위치 원칙 (`HubResponsiveWing`)
패밀리 앱에서 우측에 배치되는 공유 카드 및 파트너 배너 레이아웃은 허브 SDK(`HubResponsiveWing`)에 의해 통제됩니다.
- **[1120px 이상 (PC/태블릿 가로)]**: 우측 빈 공간에 사이드 윙(`aside`) 형태로 고정 배치합니다. 
- **[1120px 미만 (모바일/화면 축소)]**: 우측 윙을 숨기고, 화면 하단 스티키 배너 및 60초 지연 플로팅 버튼(FAB) 기반의 바텀 시트 팝업으로 대체합니다.

## 3. 공통 UI 사전 승인 원칙 (충돌 방지)
- PWA 앱 설치 팝업, 전역 로그인 모달 등은 허브 전담 AI가 통일성 있게 패밀리 앱에 전파할 의무를 가집니다.
- 단, 개별 앱의 고유한 레이아웃이나 기존 기능과 시각적 충돌이 명확히 예상되는 경우, 허브 AI는 코드를 강제 덮어쓰기 전에 **반드시 사장님(멀린)의 사전 승인을 구한 뒤** 안전하게 적용해야 합니다.

## 4. 모바일 반응형(Mobile-First) UI 기초 공사
웹앱(PWA)이 네이티브 앱처럼 동작하도록 다음 뼈대 원칙을 허브 AI가 공통 레이아웃에 주입합니다.
- 하단 스티키 바나 플로팅 버튼(FAB)에 `pb-[env(safe-area-inset-bottom)]` 적용 (기기 하단 간섭 방지).
- 모바일에서 공유 버튼 등 핵심 액션은 1분 지연 노출(Dynamic Nudge)하여 피로도를 낮춤.
