# 📬 Custom Email Notification Integration Guide

본 문서는 Merlin Hub SDK의 커스텀 이메일 발송 위임(Custom Emailing SDK) 규격에 대한 명세입니다.

---

## 1. 개요 및 동작 구조
* **동작 원리**: 개별 앱(예: 썬드리머)에서 원하는 시점(예: 로컬 포인트 변동 등)에 SDK의 `sendNotification` API를 호출하면, 허브 백엔드의 Resend / SMTP 인프라를 통해 회원 이메일로 템플릿 메일을 즉시 우회 발송합니다.
* **코인 변동 자동 메일링 분리**:
  * 코인의 차감, 충전, 가입 축하 보상 등 모든 코인 관련 증감 이벤트는 허브가 트랜잭션 성공 시 직접 감지하여 **자동으로 이메일 영수증을 발송**하므로, 개별 앱 개발자가 이를 처리할 필요가 전혀 없습니다.

---

## 2. API 호출 스펙 (SDK Method)

개별 앱 클라이언트 및 백엔드(동작 흐름에 따라)에서 SDK의 `sendNotification` 인스턴스를 통해 호출합니다.

```typescript
import { MerlinHubClient } from '@/src/services/merlin-hub-sdk/react';

const client = new MerlinHubClient();

// 자산 포인트 변동 등 필요한 시점에 이메일 전송 트리거
const res = await client.sendNotification({
  userId: "user_uuid",                 // family_users.id (유저 고유 UUID)
  title: "포인트 변동 안내",            // 메일 제목
  content: "Merlin 님, 오프라인 리워드로 100 P가 적립되었습니다.", // 메일 상세 내용
  link: "https://sundreamer.app/my-points" // [선택] 하단 바로가기 버튼 링크
});

if (res.success) {
  console.log("이메일 발송 성공:", res.email);
}
```

---

## 3. 백엔드 직접 전송 (API Endpoint)

서버 룸(Edge Function 등 백엔드 환경)에서 직접 허브의 Notification API를 찌르는 규격입니다.

* **Endpoint**: `POST https://os.sundreamer.app/api/notification/send`
* **Headers**:
  * `Content-Type: application/json`
  * `x-app-secret: {YOUR_APP_CLIENT_SECRET}`
* **Body**:
```json
{
  "userId": "user-uuid-here",
  "app_id": "SunDreamer",
  "title": "포인트 변동 안내",
  "content": "회원님의 썬드리머 포인트가 변동되었습니다.\n지급 포인트: +100 P",
  "link": "https://sundreamer.app/my-points",
  "channels": ["email"]
}
```
