# 📧 Notification Bridge Integration Guide (Email Only)

본 문서는 Merlin Hub SDK의 통합 이메일 알림 전송 브릿지에 대한 연동 명세입니다. (인앱 푸시 및 알림 종 뱃지 정책은 제거되었습니다.)

---

## 1. 알림 설정 동의 (UI/UX)

사용자는 프로필 설정 카드(`HubNotificationCard`)를 통해 이메일 알림 수신 동의 설정을 변경할 수 있으며, 이 상태는 허브 DB의 `notification_settings.email` 컬럼에 실시간으로 보관됩니다.

### 설정 카드 노출 예시
```typescript
import { HubNotificationCard } from '@/src/services/merlin-hub-sdk/react';

export function SettingsEmailAlert() {
  return (
    <HubNotificationCard
      title="이메일 설정"
      toggleLabel="📧 이메일 알림 수신"
      description="관심 채널의 신뢰도 등급 변동 시 이메일로 중요 소식을 보내드립니다."
    />
  );
}
```

---

## 2. 이메일 트리거 발송 (개별 앱 연동)

개별 패밀리 앱(어그로필터 등)은 조건 만족 시 백엔드 혹은 프론트엔드 단에서 SDK를 통해 통합 알림 발송을 요청할 수 있으며, 시스템은 사용자의 이메일 수신 동의 여부를 확인하여 Naver SMTP를 통해 메일을 발송합니다.

### 사용법 예시 (`useHubNotifier`)
```typescript
import { useHubNotifier } from '@/src/services/merlin-hub-sdk/react';

const { sendEmail } = useHubNotifier();

// 이메일 발송 요청
const handleTrigger = async () => {
  await sendEmail("TEMPLATE_ID", {
    userId: "수신유저_UUID",
    title: "영상 분석 완료 🎉",
    content: "요청하신 유튜브 영상 분석 결과가 완성되었습니다! 지금 Plaza에서 확인해 보세요.",
    link: "/p-plaza"
  });
};
```

## 3. 중요 공지사항 우회 규칙 (isNotice)

- **일반 알림 메일 발송 (기본값)**
  - 유저의 프로필 설정에 정의된 수신동의 컬럼(`notification_settings.email` 허용 여부)을 준수하여 대상 유저를 필터링 후 발송합니다.
- **중요 공지사항 발송 (`isNotice: true`)**
  - **어드민 전용 기능**으로, 긴급 서버 점검이나 장애 전파 등의 공지사항 발송 시 수신 동의 여부와 무관하게 **대상 세그먼트의 전체 유저**에게 강제 전송됩니다.
  - 이메일 발송 시 푸터 문구 역시 중요 공지용 비동의 수신문으로 자동 치환됩니다.
