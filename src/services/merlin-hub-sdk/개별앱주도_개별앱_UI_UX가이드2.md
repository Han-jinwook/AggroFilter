# [제 2 부] 개별 앱 전담 AI 자율 수행 구역 (앱 특화 UX)

아래 원칙들은 개별 패밀리 앱(어그로필터, 뭐먹지 등)을 담당하는 AI가 **해당 앱의 특성과 데이터 구조에 맞게 알아서 자율적으로 구현**해야 하는 지침입니다.

## 5. 상호작용 피드백 및 완전성 원칙 (The Click-Link Rule)
- 프로필, 설정, 취소, 메뉴 등 유저가 클릭/터치할 수 있는 **모든 인터랙티브 요소**에는 마우스 호버 시 반드시 손가락 모양 커서가 나오도록 `cursor-pointer` 클래스를 누락 없이 명시해야 합니다.
- **터치 타겟 및 마이크로 인터랙션**: 모든 클릭 가능한 요소는 44px 이상을 권장하며, 눌렀을 때 즉각적인 피드백(`active:scale-[0.98]`, `active:bg-opacity-80`)을 필수 적용합니다. 텍스트 복사가 불필요한 UI에는 `select-none`을 적용합니다.
- 기능이 연결되지 않은 껍데기 UI('먹통' 상태) 노출 금지 (반드시 Alert/Toast로 미구현 안내).

## 6. 더미 메모 온보딩 (Populated Empty State / Starter Content)
유저가 앱에 처음 진입하여 아무런 데이터가 없을 때(Blank Slate), 텅 빈 화면을 보여주는 것은 최악의 이탈을 유발합니다. 따라서 **조회 전용 페이지가 아닌, 등록(입력) 섹션이 존재하는 모든 페이지에는 무조건 페이지 특성에 맞는 '샘플(목업) 데이터'를 1개씩 만들어두어야 합니다.**
- **동작 방식**: 유저가 앱 사용법을 시각적으로 단번에 파악할 수 있도록 샘플을 노출해두고, **해당 페이지에서 유저가 실제 데이터를 단 1개라도 등록하는 순간 그 페이지의 샘플 데이터는 즉시 수거(영구 Hide)하는 개념**입니다. 전체 데이터가 아닌 철저히 '해당 페이지' 단위로 수거가 이루어져야 합니다.
- **샘플(견본)의 시각적 표시**: 유저가 본인의 데이터로 오인하지 않도록, 앱 담당 AI는 앱의 디자인 성격에 맞춰 다음 두 가지 중 하나의 방식으로 샘플임을 명시해야 합니다.
  - **[A안] 글래스모피즘 리본**: 카드 우측 상단 모서리에 반투명한 대각선 띠(`💡 SAMPLE`)를 두릅니다.
  - **[B안] 하단 페이드아웃 + 툴팁**: 카드 목록의 맨 아래를 그라데이션으로 흐리게 처리(`mask-image: linear-gradient`)하고, "첫 데이터를 입력하면 샘플은 사라져요!" 라는 안내 툴팁을 띄웁니다.
