# 어그로필터 AI 분석로직 명세 (2단계 팩트체크 선행 아키텍처)

## 1. 아키텍처 개요
본 분석 로직은 대용량 자막과 실시간 웹 검색(Google Search Grounding)이 결합될 때 발생하는 **서버리스 타임아웃(60초 초과)을 방지**하고, **분석의 논리적 일치성을 극대화**하기 위해 고안되었습니다.

사람의 논리와 동일하게 **[팩트체크 선행 ➡️ 팩트를 기반으로 요약/분석/점수 매기기]** 순서로 작동하며, 이를 안정적으로 구현하기 위해 Gemini API를 2단계로 나누어 호출합니다.

```
[영상 분석 요청]
       │
       ▼
┌──────────────────────────────────────────────┐

│ - 소요시간: 2 ~ 4초                           │
```

## 2. 세부 단계별 로직

### 1단계: 팩트체크 필요성 및 검색어 판정
- **목적**: 불필요한 검색 API 호출을 막아 속도를 높이고, 검색이 필요한 경우 최적의 키워드를 선별합니다.
- **모델**: `gemini-2.5-flash-lite` (검색 도구 비활성화)
- **입력**: 영상 제목, 채널명, 자막 데이터
- **수행 작업**:
  1. 이 영상이 객관적 사실 여부를 확인해야 하는 시사, 인물, 폭로, 정치, 최신 기술, 정보성 영상인지 판단합니다.
  2. 팩트체크가 필요한 경우, 사실 확인을 위한 가장 적합한 구글 검색어(Queries)를 1~3개 추출합니다.
- **출력 형식 (JSON)**:
  ```json
  {
    "needs_grounding": true,
    "search_queries": ["검색어1", "검색어2"]
  }
  ```

### 2단계: 최종 원샷 분석 및 채점 (본 분석)
- **목적**: 팩트를 먼저 확보하여 요약, 스포일러, 점수 간의 모순을 완벽히 제거합니다.
- **모델**: `gemini-2.5-flash`
- **구글 검색(Grounding) 분기**:
  - `needs_grounding: true`인 경우: 구글 검색 도구(`googleSearch: {}`)를 활성화하고, 1단계에서 추출된 `search_queries`를 컨텍스트에 힌트로 전달하여 자율적/정밀 검색을 유도합니다.
  - `needs_grounding: false`인 경우: 구글 검색 도구 없이 분석을 수행합니다. (검색 비용/시간 0)
- **수행 작업**:
  1. (검색 활성화 시) 검색된 실시간 정보를 바탕으로 영상 주장의 참/거짓 및 정확성을 선행 파악합니다.
  2. 제목과 내용의 차이(어그로 갭)를 분석하여 어그로 점수, 정확성 점수, 신뢰도 점수를 산정합니다.
  3. 전체 맥락 요약, 썸네일 스포일러(인용 및 출처 명시), 점수 구간별 강제 문구 규칙이 적용된 평가 사유를 원샷으로 도출합니다.
- **출력 형식**: 기존 명세된 분석 JSON 데이터 포맷을 동일하게 유지합니다.

## 3. 안정성 및 예외 처리
1. **타임아웃 방지**:
   - 1차 Lite 호출은 2~4초 내에 무조건 완료됩니다.
   - 2차 본 분석 호출 시, 검색이 활성화되더라도 1차에서 타겟팅된 명확한 범위 안에서 검색하므로 무한 루프나 지연이 발생하지 않으며, 단일 시도 timeoutMs를 45~50초로 설정하여 60초 서버리스 라이프사이클을 안전하게 보장합니다.
2. **환각 교정**:
   - 2단계 본 분석 시 실제로 구글 검색 도구가 동작하였는지(`groundingUsed`) 체크하여, 실제 검색을 수행하지 않았음에도 "검색 결과와 일치합니다"라고 거짓말하는 환각 문구를 서버 단에서 강제 필터링합니다.
3. **분석 부적합 영상 필터링**:
   - 카테고리 입구컷 및 화이트/블랙리스트 룰은 AI 호출 전에 선행 수행하여 불필요한 AI 비용 발생을 차단합니다.
